# Altogether Agile — Website Tools

A reference to the interactive tools on the site and how they are implemented.
Keep this updated when tools are added or changed.

## Stack

- **Frontend:** React + TypeScript + Vite SPA. Tailwind + shadcn/ui for newer UI
  (some older marketing pages use inline styles).
- **Backend:** Supabase (Postgres + Auth + Edge Functions), project ref
  `wqaplkypnetifpqrungv`.
- **Hosting:** Vercel; deploys on push to `main`.

## Shared patterns

These conventions are reused across the tools — follow them for new work.

- **AI** runs in **Supabase Edge Functions** (Deno) that call **Claude Sonnet 4.6**
  (`claude-sonnet-4-6`) via the shared helper
  `supabase/functions/_shared/anthropic.ts` → `callClaudeJSON(...)`. All AI
  functions are **auth-gated** (require a signed-in user). Secret:
  `ANTHROPIC_API_KEY`. Claude has no JSON mode and `claude-sonnet-4-6` rejects
  assistant-message prefill, so the helper instructs strict JSON in the system
  prompt and extracts the first balanced `{…}` object from the reply. Give AI
  calls generous `max_tokens` — Sonnet is wordier than gpt-4o-mini and will
  truncate into invalid JSON if the limit is too low.
- **Canvas tools** share `src/hooks/canvas/` (`useCanvas`, `useCanvasHistory`,
  `useCanvasInteraction` — selection, marquee box-select, group drag, zoom,
  undo/redo) and `src/utils/canvas/canvasExporter.ts` (PNG/PDF via
  html2canvas/jspdf).
- **Persistence:** tools save into a **Project** as an "artifact"
  (`project_artifacts` table: `artifact_type` + JSON `data`).
  `src/pages/ArtifactViewer.tsx` re-opens an artifact by switching on
  `artifact_type`; auto-save via `useProjectArtifactMutations`.
  `SaveToProjectDialog` handles the save/create-project flow.
- **SEO:** crawler-facing `<title>`/meta/JSON-LD is generated by
  `scripts/prerender.mjs` (the source of truth for crawlers), NOT the React
  `<Helmet>` / `SEOHead` tags. Update both when changing a page's meta.
- **Deploy edge functions:** `npx supabase functions deploy <name> --project-ref wqaplkypnetifpqrungv`
  (supabase is a devDependency; CLI logged in, project linked).

## Tools (AI Tools Hub, `/ai-tools`)

### Business Model Canvas Generator — `/bmc-generator`
AI-generates a 9-block Business Model Canvas from company details.
- Frontend: `src/components/bmc/*`, page `src/pages/BMCGenerator.tsx`.
- AI: edge fn `generate-business-model-canvas` (Claude). PDF via `fill-bmc-pdf`.
- Persistence: artifact_type `bmc`.

### User Story Canvas — `/user-story-canvas`
Interactive canvas of epic / feature / story / sticky cards with AI story
generation and INVEST-style analysis.
- Frontend: `src/components/canvas/AIToolsCanvas.tsx` (uses `useCanvasInteraction`).
- AI: edge fns `generate-user-story` (story generation) and `generate-canvas`
  (`canvasType: 'story-analysis'`), both Claude.
- Persistence: Save to Project (artifact_type `canvas`/`user_story`), auto-save.
  Export: PNG.

### Project Modelling Canvas — `/project-modelling`
Hexagonal brainstorming canvas: knowledge items, planning focuses, custom hexis,
sticky notes, artifact links.
- Frontend: `src/components/canvas/ProjectModellingCanvas.tsx` (shares
  `useCanvasInteraction`).
- AI: none — manual; pulls knowledge items from Supabase.
- Persistence: artifact_type `project-model`, auto-save. Export: PNG/PDF.

### Product Backlog — `/backlog`
Capture, prioritise, and drag-and-drop backlog items.
- Supabase-backed, manual (no AI). artifact_type `product-backlog`.

### Impact Map Builder — `/impact-map`
Guided Impact Mapping (Gojko Adzic): a structured top-down editor through
Goal (Why) → Actors (Who) → Impacts (How) → Deliverables (What), with coaching
text and per-level AI suggestions.
- Frontend: `src/components/impactMap/ImpactMapEditor.tsx`,
  `src/types/impactMap.ts`, `src/utils/impactMap/exportImpactMap.ts`.
- AI: edge fn `generate-impact-map` (Claude) — "Suggest (AI)" at each level.
- Exports: FreeMind `.mm`, PNG, PDF, JSON, Markdown; JSON import.
- Persistence: localStorage autosave (standalone) + Save to Project
  (artifact_type `impact-map`); when opened from a project artifact it
  auto-saves back to the artifact.

### Coaching Studio — `/coach`
A standalone coached conversation (not tied to a cell), the pipeline's front door.
Free chat via `coach-reflect` (`session` mode), then **harvest**: `coach-harvest`
reads the transcript and proposes where each thing lands (goal, backlog, probe,
benefit, persona, agreement, note) in the person's own words. Send-to-backlog
writes a real `backlog_items` row when saved in a project; other destinations open
the tool and stay recorded in the session.
- Frontend: `src/components/coachingStudio/CoachingStudioEditor.tsx`,
  `src/types/coachingSession.ts`.
- Persistence: Save to Project (artifact_type `coaching-session`: transcript +
  harvested items + where they went).

### Persona Studio — `/personas`
Coached persona (role, context, goals, pains, behaviours, voice) plus an avatar
image (public `user-uploads` bucket). Exports PNG/PDF/JSON/Markdown.
`src/components/persona/PersonaEditor.tsx`; artifact_type `persona`.

### Canvas Catalogue — `/canvases` and `/canvases/:canvasKey`
Data-driven coached-canvas engine (`src/config/canvases.ts` +
`CoachedCanvasEditor`). Business Model Canvas, Business Case, Product Vision, plus
a Canvas Picker. artifact_types `business-case`, `product-vision` (BMC via `bmc`).

### Probe Tracker — `/probes`
Experiment kanban for the Outputs stage: each output option is an
option-as-hypothesis card moving Planned → Running → Kept/Killed, with the
smallest test and the signal that would prove it wrong. Coached on the test.
`src/components/probeTracker/ProbeTrackerEditor.tsx`; artifact_type `probe-tracker`.

### Benefits Scorecard — `/benefits`
Outcomes and Value stage: each outcome has a leading indicator, target and dated
readings, with an inline trend sparkline and a Benefits on a Page PDF export.
`src/components/benefitsScorecard/BenefitsScorecardEditor.tsx`; artifact_type
`benefits-scorecard`.

### Ways of Working — `/ways-of-working`
Operate stage: working agreements plus short coached retrospectives that land on
one improvement action at a time (standing stretch displayed).
`src/components/waysOfWorking/WaysOfWorkingEditor.tsx`; artifact_type
`ways-of-working`.

The Hub and each project page open with a six-stage **journey view**
(`src/components/pipeline/JourneyBand.tsx` and `ProjectJourney.tsx`), driven by the
Pipeline Registry (`src/config/pipeline.ts`).

## Adjacent features (not in the Hub)

### Practice Exams — `/exams`
AgilePM Foundation and Professional Scrum Master mock exams (Supabase `exams`
table, `ExamPlayer`). Timed and revision modes. No AI. Public.

### Knowledge Base + Pattern Builder — `/knowledge-base`
Built and in production, but **unpublished** — gated by `show_knowledge = false`
in `site_settings` (hidden from the public menu / 404s until flipped). Pattern
Builder uses edge fns `recommend-pattern` (Anthropic/Claude, a direct call that
predates the shared helper) and `pattern-builder-feedback` (no LLM; records
thumbs up/down feedback). See the ISA-O3 Knowledge Base build notes for detail.

on 2026-06-10 and verified working end-to-end (signed-in):
